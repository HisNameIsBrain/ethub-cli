# ETHUB CLI: Complete Documentation & Benchmarks

## 1. Core Architecture & Function Breakdown

### `HybridEngine` (core/hybrid_engine.py)
*The central orchestration layer merging reasoning, local knowledge, and action planning.*
- **`process_query(query, messages)`**
  - **Function:** Analyzes a query, retrieves relevant internal training data via RAG, and plans execution strategy.
  - **Output:** Returns a tuple `(reasoning_dict, action_dict)`.
- **`get_local_knowledge(query, top_k)`**
  - **Function:** Performs a keyword-based retrieval on `agent-data/training.json`.
  - **Output:** A concatenated string of the most relevant internal memory snippets.

### `ReasoningEngine` (core/reasoning_engine.py)
*The intent classification module.*
- **`analyze(query)`**
  - **Function:** Queries the local LLM (Ollama) to classify the user's intent (e.g., `code_query`, `hardware_query`, `general_query`) and extracts key parameters.
  - **Output:** A JSON object containing `"query_type"`, `"intent"`, and `"keywords"`.

### `ActionEngine` (core/action_engine.py)
*The strategy planning module.*
- **`decide_actions(reasoning)`**
  - **Function:** Determines the optimal search sources (e.g., `github`, `stackoverflow`, `npm`) based on the reasoning payload.
  - **Output:** A JSON object defining the planned action and targeted data sources.

### `HelperEngine` (core/helper_engine.py)
*Handles logging, output formatting, and dashboard synchronization.*
- **`log_live(category, message, details)`**
  - **Function:** Appends real-time events to `agent-data/live_logs.json` for dashboard syncing.
- **`log_console(text)`**
  - **Function:** Captures raw CLI output for the web dashboard mirror.

### `WebServer` (core/web_server.py)
*The local telemetry and debugging dashboard.*
- **`start_server(port)`**
  - **Function:** Initializes a threaded local HTTP server serving the dashboard UI and API endpoints.
  - **Output:** Background server process on specified port (default: 8080).


## 2. How-To & Practical Examples

### Starting the Interactive Mode
To run ETHUB CLI with the live web dashboard active:
```bash
python3 ethub_cli.py --interactive
```

### Performing an Autonomous Search
```bash
python3 ethub_cli.py "How do I reverse an array in Rust?"
```
**Execution Flow Example:**
1. **Reasoning:** Identifies intent as `code_query` and keywords `["reverse", "array", "rust"]`.
2. **Action:** Targets `github` and `stackoverflow`.
3. **Execution:** Performs targeted searches, fetches URLs, audits them via `SafetyEngine`, and synthesizes a final answer.

### Modifying Settings in Real-time
You can change the active model on the fly using internal CLI commands:
```text
ETHUB> /set model llama3
[SUCCESS] Setting 'model' updated to 'llama3'
```
