# ETHUB CLI: Complete Documentation & Benchmarks (v2.1 Surgical)

## 1. Core Architecture & Function Breakdown

### `SurgicalActionEngine` (core/surgical_engine.py)
*The high-integrity filesystem interaction layer.*
- **`apply_patch(target, content)`**
  - **Function:** Applies a surgical code fix with automated SHA256 integrity verification.
  - **Integrity:** Automatically creates a `.bak` snapshot in `.ethub/snapshots/` before writing.
- **`list_files()` & `read_target(f)`**
  - **Function:** Performs directory-aware context gathering with SHA256 metadata.

### `EthubReturnEngine` (core/return_engine.py)
*The unified system recovery and restoration system.*
- **`capture_point(label)`**
  - **Function:** Captures a point-in-time manifest of all agent state (history, memory, training) for future restoration.
- **`restore_to(point_id)`**
  - **Function:** Reverts the entire agent environment to a specific point in time.
- **`rollback_surgical_fix(file_name)`**
  - **Function:** Reverts an individual file to its latest surgical backup.

### `HybridEngine` (core/hybrid_engine.py)
*The central orchestration layer merging reasoning, local knowledge, and action planning.*
- **`process_query(query, messages)`**
  - **Function:** Analyzes intent, retrieves local knowledge (RAG), and plans the optimal surgical action.
- **`get_local_knowledge(query, top_k)`**
  - **Function:** Performs keyword-based retrieval on `agent-data/training.json`.

### `ReasoningEngine` (core/reasoning_engine.py)
*The intent classification module.*
- **`analyze(query)`**
  - **Function:** Uses a local LLM to classify intent into categories like `surgical_fix`, `return_recovery`, or `code_query`.

### `SafetyEngine` (core/safety_engine.py)
*The 9-point security firewall.*
- **`perform_surgical_audit(fix_cmds, diff_patch)`**
  - **Function:** Audits proposed fixes for dangerous patterns (e.g., `rm -rf /`, `/etc/shadow`).
- **`audit_source_chunk(query, source, chunk)`**
  - **Function:** Verifies the safety of inbound web content.

### `HelperEngine` (core/helper_engine.py)
*Handles console formatting, UI boxes, and system information.*

### `WebServer` (core/web_server.py)
*Hosts the ARIA-compliant Surgical Management Dashboard.*


## 2. Practical Examples & Workflows

### Starting the Surgical Console
```bash
python3 ethub_cli.py --interactive
```

### Executing a Surgical Fix Workflow
1. **Analyze**: Agent identifies a bug in `main.py`.
2. **Research**: Agent uses `web_search` and `fetch_url` to find a resolution.
3. **Draft**: Agent proposes a `DIFF_PATCH` and `FIX_CMDS`.
4. **Audit**: `SafetyEngine` verifies the fix is surgically sound.
5. **Execute**: User (or agent) applies the fix via `ethub_action`.
6. **Recovery**: A snapshot is created in `.ethub/snapshots/` for a potential rollback.

### Capturing a System Return Point
```text
ETHUB> /return capture "Pre-Deployment Snapshot"
[SUCCESS] Return point captured: 20260308_150000
```
